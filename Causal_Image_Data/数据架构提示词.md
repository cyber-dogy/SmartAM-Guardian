# SLM增材制造多模态异常识别数据集 - 数据架构说明

## 1. 数据集概述

这是一个**选择性激光熔化（SLM）增材制造过程的多模态异常识别数据集**，用于训练机器学习模型识别金属3D打印过程中的工艺异常。

- **任务类型**：多模态图像分类（正常 vs 异常，及异常类型识别）
- **核心目标**：基于多视角视觉数据识别激光选区熔化过程中的工艺缺陷
- **数据划分原则**：按工况组（condition_id）划分，同一工况的所有层级样本必须整体划分到同一集合

---

## 2. 数据组织层级

```
Causal_Image_Data/
├── metadata.csv          # 主元数据文件，包含所有样本信息
├── images/               # 图像数据目录
│   └── {condition_id}/   # 工况组文件夹
│       └── L{layer_id:04d}/   # 层级样本文件夹
│           ├── rgb_view1.jpg      # CH1: 正面斜上方可见光
│           ├── rgb_view2.jpg      # CH2: 侧后上方可见光  
│           ├── ir_before.jpg      # CH3: 红外相对温度图（打印后）
│           └── ir_after.jpg       # CH3: 红外相对温度图（铺粉后）
```

### 2.1 层级关系
- **Condition（工况组）**：对应唯一的工艺参数组合，如 `Base-01`, `High-02`
- **Layer（层级样本）**：同一工况下的单个打印层，包含4张多模态图像
- **重要规则**：同一Condition下的所有Layer必须整体划分，不能分散到不同数据集

---

## 3. 元数据字段定义（metadata.csv）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `condition_id` | string | 工况唯一标识，如 `Base-01`, `High-02`, `Low-04` |
| `layer_id` | int | 层号，从1开始递增（注意：实际采集可能存在跳层） |
| `power_w` | int | 激光功率 P (W)，如 120, 240, 300 |
| `speed_mms` | int | 扫描速度 V (mm/s)，如 500, 1000, 2000 |
| `spacing_mm` | float | 填充间距 S (mm)，如 0.05, 0.08, 0.10 |
| `scan_strategy` | string | 扫描策略，`Stripe` 或 `Zigzag` |
| `energy_density` | float | 面能量密度 Ed = P/(V×S)，单位 J/mm² |
| `rgb_view1_path` | string | CH1图像原始路径（正面斜上方视角） |
| `rgb_view2_path` | string | CH2图像原始路径（侧后上方视角） |
| `ir_view_before_path` | string | CH3_before图像原始路径（打印后制件表面） |
| `ir_view_after_path` | string | CH3_after图像原始路径（铺粉后粉层表面） |
| `batch_id` | int | 批次标识，重复打印组标记为2 |
| `role` | string | 工况角色，见下方角色定义 |
| `defect_label_binary` | int | 二分类标签：0=正常, 1=异常 |
| `defect_label_type` | string | 异常类型：NOR/HEW/LEL |

## 4. 工艺参数与角色体系

### 4.1 工况角色 (role)

| 角色值 | 含义 | 用途说明 |
|--------|------|----------|
| `source_train` | 基准训练源域 | 用于构建基础正常样本分布 |
| `source_repeat` | 基准重复验证 | 验证批次稳定性和数据可靠性 |
| `causal_intervention_high` | 高能因果干预 | 激光功率↑导致的过烧/翘曲异常（必做异常识别） |
| `causal_intervention_low` | 低能因果干预 | 激光功率↓导致的未熔合异常（必做异常识别） |
| `causal_equiv_test` | 等效能量验证 | Ed相同但参数不同的组合，测试模型是否真正理解物理机制 |
| `unseen_strategy_shift` | 未见扫描策略 | 训练时用Stripe，测试时用Zigzag，验证策略迁移能力 |
| `unseen_texture_shift` | 未见纹理偏移 | 仅改变间距S导致的视觉纹理变化，测试虚假异常判别 |
| `unseen_param_shift` | 未见参数偏移 | 速度变化导致的能级偏移 |
| `unseen_composite_shift` | 未见复合偏移 | 多个维度同时变化的复合场景 |

### 4.2 标签体系

**二分类标签 (`defect_label_binary`)：**
- `0` = 正常/可接受 (normal)
- `1` = 异常 (abnormal)

**异常类型标签 (`defect_label_type`)：**
- `NOR` = Normal，正常样本
- `HEW` = High Energy Warping，高能导致的翘曲/过烧类异常
- `LEL` = Low Energy Lack of fusion，低能导致的未熔合类异常

**标签分配原则：**
- 基准组 (`Base`) → NOR
- 明显高能异常（P≥240，Ed≥3.0） → HEW
- 明显低能异常（P≤85，Ed≤1.06） → LEL
- 等效能量组 (`Equiv`) → NOR（用于机理验证）
- 纯纹理偏移（仅S变化，Ed正常范围） → NOR

## 5. 模态说明与使用建议

### 5.1 四模态数据

| 模态 | 文件名 | 视角/内容 | 特点与建议 |
|------|--------|-----------|------------|
| RGB View 1 | `rgb_view1.jpg` | 正面斜上方可见光 | 主要推荐用于建模，清晰的制件表面信息 |
| RGB View 2 | `rgb_view2.jpg` | 侧后上方可见光 | 辅助视角，提供空间深度信息 |
| IR Before | `ir_before.jpg` | 打印后制件表面红外图 | 相对温度颜色映射，反映热积累历史 |
| IR After | `ir_after.jpg` | 铺粉后粉层表面红外图 | 可能受激光扫描光斑干扰导致特征失真，慎用 |

### 5.2 模态融合策略建议

```python
# 推荐的多模态输入组合
primary_modalities = [rgb_view1, rgb_view2]      # 主要依赖可见光
thermal_modalities = [ir_before]                  # 辅助热信息（相对温度）
# ir_after 因激光干扰问题，建议仅在特定场景下使用或进行预处理
```

## 6. 关键约束与注意事项

### 6.1 数据划分约束（⚠️ 必须遵守）

```python
# 错误做法 ❌
train_samples = random_sample(all_layers, ratio=0.8)
test_samples = remaining_layers
# 这会导致同一condition的层被分到不同集合，造成数据泄露！

# 正确做法 ✅
train_conditions = ['Base-01', 'High-02', 'Low-02', ...]
test_conditions = ['High-05', 'Gen-01', ...]
# 以condition为单位整体划分！
```

**核心规则：同一 `condition_id` 下的所有 `layer_id` 必须整体分配到同一数据集（train/val/test）**

### 6.2 层间时序关系

- 同一Condition内的 `layer_id` 具有时序相关性（层间存在热积累效应）
- 若做时序建模，应保留层顺序；层号可能不连续（如 L0003→L0004→L0008）
- 静态识别也应考虑：相邻层不是严格独立样本

### 6.3 红外数据解释

- CH3红外图像是**相对温度颜色映射**，不是绝对温度矩阵
- 不应直接按像素绝对值解释温度，建议归一化处理或作为纹理特征

### 6.4 能量密度参考值

| 类别 | Ed 范围 | 典型表现 |
|------|---------|----------|
| 低能区 | < 1.0 | 明显未熔合风险 |
| 低能过渡 | 1.0 - 1.5 | 轻微未熔合倾向 |
| 正常区 | 1.5 - 2.0 | 正常成形 |
| 高能过渡 | 2.0 - 3.0 | 轻微过烧倾向 |
| 高能区 | > 3.0 | 明显过烧/翘曲风险 |

## 8. 建模任务定义

### 8.1 主任务：二分类异常识别
- 输入：多模态图像（推荐 RGB_V1 + RGB_V2 + IR_Before）
- 输出：0=正常, 1=异常
- 评估：Accuracy, Precision, Recall, F1, AUC-ROC

### 8.2 进阶任务：异常类型识别
- 输入：同上
- 输出：NOR / HEW / LEL（三分类）
- 评估：Confusion Matrix, Per-class F1

### 8.3 泛化任务：域迁移测试
- 训练域：`source_train` + `causal_intervention_*`
- 测试域：`unseen_*` 角色组
- 目标：验证模型在未见扫描策略、未见纹理、未见复合工况下的迁移能力

### 8.4 因果稳定性任务
- 对比：`causal_equiv_test` 组 vs 真实异常组
- 验证：模型是否真正理解物理机制，而非仅记忆表面特征
